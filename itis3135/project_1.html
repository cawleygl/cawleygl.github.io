Project 1

Project Overview

This project website will be an interactive product backlog for Mary Dare Martin's Stuff Made by MD storefront. Stuff Made By MD is an online store that sells a variety of handmade gifts through Etsy, consistently adding new items and rotating out old ones. This website will primarily consist of interactive product pages, where all inventory items can be searched and selected individually. Items that are currently for sale will include their sale price and a link to purchase, while unavailable items will display the item's status (seasonal, out of stock, or discontinued) to explain why they are currently not available for purchase. Shoppers of Stuff Made By MD are the intended users of this site, and here they will be able to confirm the status of items that may have appeared on the store in the past, browse to find new products that they might be interested in, and leave reviews of products that will remain even after items must be removed from the store.

Site Map

Landing Page

  • Purpose: The purpose of the landing page is to communicate the company's mission and branding, while suggesting a new product to the user upon their arrival. One featured item will be presented to the user on this page.
  • Audience: Customer
  • Content: Company name, other branding, and a featured item preview.
  • Is this a page where you are asking users to enter data on the page? None
  • Will those data fields need any validations (if any)? N/A
  • Buttons, Hyperlinks, or Dropdowns: This page will contain a preview of one product, which acts as a link to that item's product page. It will also contain a purchase link for the item.
  • Actions: When the user clicks on the featured item, it will take them to that item's product page.
  • Notes: The company logo on the left side of the navbar will redirect back to this page.

Featured Items Gallery Page

  • Purpose: The purpose of the featured items gallery page is to showcase a number of featured, currently available items to the user. It will use a gallery format to present a detailed image and a complete description to the user, and allow navigation using arrow buttons.
  • Audience: Customer
  • Content: Large product image, lengthy product description, purchase link, arrow controls to cycle through featured products.
  • Is this a page where you are asking users to enter data on the page? None
  • Will those data fields need any validations (if any)? N/A
  • Buttons, Hyperlinks, or Dropdowns: The page will contain arrow buttons to cycle through previews of the featured products. The previews will contain purchase links, and links to the items' product pages.
  • Actions: When the user clicks on the arrow buttons, the gallery will navigate between featured items.
  • Notes: A set of available items will periodically be designated as “featured.” One featured item will be showcased on the landing page, while all featured items will appear on the gallery page.

Product Page

  • Purpose: The purpose of the individual product pages is to allow the user to view definitive information about every item the store has to offer, and suggest a few related items to the user.
  • Audience: Customer
  • Content: Full-sized product image, complete product description, purchase link, list of small previews of related items.
  • Is this a page where you are asking users to enter data on the page? None
  • Will those data fields need any validations (if any)? N/A
  • Buttons, Hyperlinks, or Dropdowns: This page will contain a purchase link if the item is available for purchase, and previews of related items, which act as links to these items pages.
  • Actions: Links to store page if item is available.
  • Notes: Clicking on any item preview from other pages redirects to the item's product page.

Search Page

  • Purpose: The purpose of the search page is to allow the user to find items using a keyword search. This will function as a quick and easy way to check a specific item's availability, or quickly navigate to a specific item's product page for more information.
  • Audience: Customer
  • Content: Search text input, list of item results only containing the items' names and availability. Clicking a result will redirect the user to that item's product page.
  • Is this a page where you are asking users to enter data on the page? Yes
  • Will those data fields need any validations (if any)? No
  • Buttons, Hyperlinks, or Dropdowns: Search results will act as links to product pages.
  • Actions: Typing into the search bar will populate a list of items.

About Page

  • Audience: Customer
  • Content: Image of the store's founder Mary Dare Martin, paragraphs about the store's mission and history, and links to the store's official social media pages.
  • Is this a page where you are asking users to enter data on the page? No
  • Will those data fields need any validations (if any)? N/A
  • Buttons, Hyperlinks, or Dropdowns: Direct links to the Etsy storefront and official store social media pages.
  • Actions: Hyperlinks will take the user to sites with further information about Stuff Made by MD.

Dynamic Functionality

Many of the pages in this website will include interactive elements using Javascript. The search function on the search page will use Javascript to select and display a list of items that contain the entered keyword in its name or description. The product pages will use Javascript to dynamically load information about an item, while maintaining an identical page design and structure. Online retail sites, like Amazon, Ebay, and many others use this format in their online shopping sites. The gallery functionality on the featured items page will use Javascript to cycle through a set of featured items, and will use Javascript to periodically select which items are designated as featured. Pexels is a good example of a site that uses a full-screen gallery preview when viewing options.
